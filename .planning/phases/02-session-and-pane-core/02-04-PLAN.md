  <name>Task 1: Extend prefix key dispatcher with pane navigation, resize, split, and kill keybindings</name>
  <files>src/ipc/client.rs, src/wt.rs</files>
  <action>
1. In `src/ipc/client.rs`, extend the prefix key state machine from Plan 02's attach function. After Ctrl+B is pressed and prefix_active is true, handle these additional key sequences:

   **Pane Navigation (Prefix + arrow key):**
   - Arrow keys in raw mode are sent as escape sequences: ESC [ A (up), ESC [ B (down), ESC [ C (right), ESC [ D (left)
   - When prefix_active and input starts with ESC (0x1B):
     - Read the full escape sequence (ESC [ {letter})
     - Map to direction: A=Up, B=Down, C=Right, D=Left
     - Call `wmux::wt::wt_move_focus(direction)` to tell WT to focus adjacent pane
     - Also send `Request::NavigatePane { session_id, direction }` to daemon so it can update active_pane tracking
     - Reset prefix_active

   **Pane Resize (Prefix + Alt+arrow):**
   - Alt+arrow in raw mode: ESC ESC [ A (or ESC [ 1 ; 3 A for Alt modifier)
   - When prefix_active and detect Alt+arrow sequence:
     - Call `wmux::wt::wt_resize_pane(direction, amount)` with a default resize amount (e.g., 5 cells)
     - Send `Request::ResizePane` to daemon to update ConPTY size via ResizePseudoConsole
     - Reset prefix_active

   **Split keybindings (Prefix + " and Prefix + %):**
   - When prefix_active and byte == '"' (0x22): trigger horizontal split
     - Same logic as `wmux split -h`: send SplitPane request, then wt_split_pane
   - When prefix_active and byte == '%' (0x25): trigger vertical split
     - Same logic as `wmux split -v`: send SplitPane request, then wt_split_pane

   **Kill pane (Prefix + x):**
   - When prefix_active and byte == 'x' (0x78):
     - Print confirmation prompt at bottom: "kill-pane? (y/n)"
     - Read single byte — if 'y', send `Request::KillPane` to daemon
     - If 'n', resume normal operation

   **Detach (Prefix + d) — already from Plan 02**

   Create a helper function to organize this:
   ```rust
   async fn handle_prefix_key(byte: u8, escape_seq: Option<&[u8]>, session_id: &str, pipe: &mut ...) -> PrefixAction
   ```
   where `PrefixAction` is enum { Detach, Handled, Forward(Vec<u8>) }

2. Update `src/wt.rs` with new WT command helpers:
   - `pub fn wt_move_focus(direction: &str) -> Result<()>` — runs `wt.exe -w 0 move-focus --direction {direction}` where direction is "up", "down", "left", "right"
   - `pub fn wt_resize_pane(direction: &str, amount: u32) -> Result<()>` — runs `wt.exe -w 0 resize-pane --direction {direction} --amount {amount}`
   - Note: WT's move-focus and resize-pane are experimental features but available in recent WT versions. If not available, fall back gracefully with a warning.
