   - `pub fn is_windows_terminal() -> bool` — check if `WT_SESSION` environment variable is set (WT sets this for all processes running inside it). This is the standard detection method.
   - `pub fn require_windows_terminal() -> Result<()>` — returns `Ok(())` if inside WT, otherwise returns error: "wmux requires Windows Terminal. Please run wmux inside Windows Terminal (wt.exe)."
   - `pub fn wt_split_pane(direction: &str, command: &str) -> Result<()>` — spawns `wt.exe split-pane --{direction} {command}` as a subprocess. `direction` is "horizontal" or "vertical". Returns error if wt.exe not found.
   - `pub fn wt_focus_pane(pane_index: u32) -> Result<()>` — spawns `wt.exe focus-pane --target {pane_index}`.

2. Update `src/lib.rs` to add `pub mod wt;`.

3. Update `src/cli.rs` Commands enum:
   - `Attach { session_id: Option<String> }` — optional session ID (attaches to most recent if omitted)

   - `Split { #[arg(short = 'h', long = "horizontal")] horizontal: bool, #[arg(short = 'v', long = "vertical")] vertical: bool }` — at least one of -h or -v required
   - `KillPane { #[arg(long)] pane_id: Option<u32> }` — kills active pane if no ID given
   - `KillSession { id: String }` — unchanged
   - Keep all existing commands
