  <action>
1. Create `src/session/scrollback.rs`:
   ```rust
   pub struct ScrollbackBuffer {
       lines: Vec<Vec<u8>>,    // Ring buffer of raw line bytes
       capacity: usize,         // Max lines (default 10000)
       head: usize,             // Write position (next slot to write)
       count: usize,            // Current number of lines stored
   }
   ```

   Methods:
   - `pub fn new(capacity: usize) -> Self` — create empty buffer
   - `pub fn push_line(&mut self, line: Vec<u8>)` — add a line, evicting oldest if at capacity. Write to `lines[head]`, advance head modulo capacity.
   - `pub fn push_bytes(&mut self, data: &[u8])` — split raw byte data on newline boundaries (0x0A), push each line. Handle partial lines: buffer trailing bytes that don't end with newline, prepend to next push.
   - `pub fn line_count(&self) -> usize` — current number of stored lines
   - `pub fn get_line(&self, index: usize) -> Option<&[u8]>` — get line by absolute index (0 = oldest)
   - `pub fn get_lines(&self, start: usize, count: usize) -> Vec<&[u8]>` — get a range of lines for display
   - Internal: `fn physical_index(&self, logical_index: usize) -> usize` — converts logical (oldest=0) to physical ring buffer index

   Add a `partial_line: Vec<u8>` field for buffering incomplete lines from push_bytes.

2. Update `src/session/mod.rs` to declare `pub mod scrollback;`.

3. Update `src/session/pane.rs`:
   - Add `scrollback: ScrollbackBuffer` field to Pane struct
   - Initialize with `ScrollbackBuffer::new(10_000)` in Pane::new()
   - Add `pub fn scrollback(&self) -> &ScrollbackBuffer` and `pub fn scrollback_mut(&mut self) -> &mut ScrollbackBuffer`

4. Write tests in `tests/scrollback_test.rs` covering the behaviors listed above.

  <name>Task 2: Integrate scrollback capture in attach loop and implement scroll mode</name>
  <files>src/ipc/server.rs, src/ipc/client.rs, src/session/manager.rs</files>
  <action>
1. Update `src/ipc/server.rs` attach handler (the output streaming loop from Plan 02):
   - After reading bytes from ConPTY output, BEFORE sending to client, also push the bytes into the pane's scrollback buffer:
     ```rust
     // In the output streaming loop:
     let bytes_read = conpty.read_output(&mut buf).await?;
     // Capture in scrollback (need to lock manager briefly)
     {
         let mut mgr = session_manager.lock().await;
         if let Some(session) = mgr.get_session_mut(&session_id) {
             if let Some(pane) = session.get_active_pane_mut() {
                 pane.scrollback_mut().push_bytes(&buf[..bytes_read]);
             }
         }
     }
     // Send to client as before
     ```
   - NOTE: This means scrollback captures output even when no client is attached (the daemon's output read loop should run regardless). If the current architecture only reads ConPTY when a client is attached, consider adding a background output drain task per pane that:
     a. Continuously reads ConPTY output
     b. Pushes to scrollback buffer
     c. Broadcasts to any attached clients
   - This is important for SESS-06 (sessions persist after disconnect) — output must be captured even when detached.

2. Handle `Request::EnterScrollMode` in server:
   - Lock session_manager, get the pane's scrollback buffer
   - Send the last N lines (where N = terminal rows) as `Response::SessionOutput` so client can render the scrollback view
   - Track that this pane is in scroll mode (add `scroll_mode: bool` to Pane or track in server)

3. Handle `Request::ScrollBack { lines }` in server:
   - Get requested lines from scrollback buffer using `get_lines(offset, count)`
   - Send as `Response::SessionOutput` with the line data
   - Positive lines = scroll up (earlier in buffer), negative = scroll down (later in buffer)

4. Update `src/ipc/client.rs` attach loop — add scroll mode to prefix key handler:
   - Prefix + '[' (0x5B): Enter scroll mode
     - Send `Request::EnterScrollMode` to daemon
     - Switch client to scroll mode state where:
       - Normal keystrokes are NOT forwarded to shell
       - Arrow Up / Page Up: send `Request::ScrollBack { lines: -1 }` (or -page_size)
       - Arrow Down / Page Down: send `Request::ScrollBack { lines: 1 }` (or +page_size)
       - 'q' (0x71): exit scroll mode, send `Request::ExitScrollMode`, resume normal attach I/O
     - Render scroll position indicator (e.g., "[42/1000]" showing current line / total lines)

5. Mouse wheel support during scroll mode:
   - In raw console mode, mouse wheel events appear as console input records (INPUT_RECORD with MOUSE_EVENT)
   - Enable mouse input via SetConsoleMode with ENABLE_MOUSE_INPUT
   - In scroll mode, translate MOUSE_WHEELED events to scroll up/down requests
   - Outside scroll mode, mouse events are passed through to ConPTY normally

6. Add `pub fn get_session_pane_scrollback(&self, session_id: &str, pane_id: u32) -> Option<&ScrollbackBuffer>` to SessionManager for convenient access.
