src/daemon/recovery.rs:

Define persisted state schema:
```
#[derive(Debug, Serialize, Deserialize)]
pub struct PersistedState {
    pub version: u32,  // schema version, start at 1
    pub sessions: Vec<PersistedSession>,
    pub next_id: u32,
    pub saved_at: String,  // ISO 8601
}

#[derive(Debug, Serialize, Deserialize)]
pub struct PersistedSession {
    pub id: String,
    pub name: Option<String>,
    pub pid: u32,        // child process PID
    pub created_at: String,
    pub shell: String,   // e.g., "powershell.exe"
    pub cols: i16,
    pub rows: i16,
}
```

`pub fn save_state(state: &PersistedState) -> Result<()>`:
- Serialize to JSON with pretty printing (debuggable).
- Write to state.json.tmp first, then std::fs::rename to state.json (atomic on Windows when same directory).
- Log "State saved: {n} sessions" at debug level.

`pub fn load_state() -> Result<PersistedState>`:
- If state.json doesn't exist, return default empty state.
- If JSON parsing fails, log warning "Corrupted state file, starting fresh", rename broken file to state.json.bak, return empty state.
- Return parsed state.

`pub fn recover_sessions(persisted: &PersistedState, manager: &mut SessionManager) -> Result<RecoveryReport>`:
- For each PersistedSession:
  1. Check if PID is still alive (OpenProcess + WaitForSingleObject with 0 timeout).
  2. If alive: attempt to re-adopt. Since we can't re-attach to an existing ConPTY after daemon restart (ConPTY handles are process-local), we note it as "orphaned but alive" — the process keeps running but we cannot control it. Create a new ConPTY session for this slot and log "Session {id}: original process alive (pid {pid}), spawned replacement shell".
  3. If dead: spawn a new shell via ConPtySession::new() with the same cols/rows. Log "Session {id}: process dead, spawned new shell".
  4. Restore the session ID and name (don't generate new ones).
- Return RecoveryReport { recovered: n, respawned: m, failed: k }.

Note on re-adoption limitation: ConPTY pseudo-console handles are process-local and cannot be inherited across daemon restarts. The pragmatic approach is to respawn shells and log what happened. This matches the CONTEXT.md decision that scrollback is lost on crash. Document this limitation clearly in the code.

Update src/daemon/mod.rs to include recovery module.

Write tests in tests/recovery_test.rs for serialization round-trips and load_state edge cases (missing file, corrupted file).
