src/session/conpty.rs — ConPTY wrapper:

`pub struct ConPtySession`:
- Fields: pseudo_console handle (HPCON), process handle, thread handle, process_id (u32), pipe_in (write end for input to shell), pipe_out (read end for output from shell).
- `pub fn new(cols: i16, rows: i16, shell: Option<&str>) -> Result<Self>`:
  1. Create two pairs of pipes using CreatePipe (input pair + output pair).
  2. Create a COORD size = { X: cols, Y: rows }.
  3. Call CreatePseudoConsole(size, input_read_handle, output_write_handle, 0, &mut hpc). This gives us an HPCON.
  4. Initialize a STARTUPINFOEXW with the pseudo console handle via InitializeProcThreadAttributeList + UpdateProcThreadAttribute with PROC_THREAD_ATTRIBUTE_PSEUDOCONSOLE.
  5. Call CreateProcessW with the shell path (default: "powershell.exe", fallback to "cmd.exe"), using the STARTUPINFOEXW. Use CREATE_UNICODE_ENVIRONMENT flag.
  6. Close the pipe ends that belong to the child (input_read, output_write) — daemon keeps input_write and output_read.
  7. Store handles and return ConPtySession.
- `pub fn process_id(&self) -> u32` — return child PID.
- `pub fn is_alive(&self) -> bool` — WaitForSingleObject with 0 timeout, check if process still running.
- `pub fn kill(&mut self) -> Result<()>` — TerminateProcess on child, ClosePseudoConsole on HPCON, close all handles.
- Implement Drop to clean up handles (ClosePseudoConsole, CloseHandle on process/thread handles, close pipe handles).

Use these windows-rs APIs:
- `windows::Win32::System::Console::{CreatePseudoConsole, ClosePseudoConsole, HPCON, COORD}`
- `windows::Win32::System::Threading::{CreateProcessW, InitializeProcThreadAttributeList, UpdateProcThreadAttribute, STARTUPINFOEXW, PROCESS_INFORMATION, EXTENDED_STARTUPINFO_PRESENT, CREATE_UNICODE_ENVIRONMENT, PROC_THREAD_ATTRIBUTE_PSEUDOCONSOLE}`
- `windows::Win32::System::Pipes::CreatePipe`
- `windows::Win32::Foundation::{CloseHandle, HANDLE}`

src/session/manager.rs — Session tracking:

`pub struct SessionManager`:
- Fields: sessions: HashMap<String, Session>, next_id: u32.
- `Session` struct: id (String), conpty (ConPtySession), created_at (SystemTime).
- `pub fn new() -> Self`
- `pub fn create_session(&mut self, name: Option<String>) -> Result<SessionInfo>`:
  - Generate ID (incrementing counter, formatted as string).
  - Create ConPtySession::new(120, 30, None) with default size.
  - Store in sessions map.
  - Return SessionInfo { id, name, created_at }.
- `pub fn list_sessions(&self) -> Vec<SessionInfo>` — iterate sessions, return info.
- `pub fn kill_session(&mut self, id: &str) -> Result<()>` — find session, call conpty.kill(), remove from map.
- `pub fn session_count(&self) -> u32`.
- `pub fn get_session(&self, id: &str) -> Option<&Session>`.

src/session/mod.rs — pub mod conpty; pub mod manager; re-export SessionManager.

Note: SessionManager needs to be shared between the daemon main loop and IPC handlers. Wrap it in `Arc<Mutex<SessionManager>>` (tokio::sync::Mutex for async contexts) and pass to the ControlServer.
