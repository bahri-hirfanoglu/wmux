src/daemon/lifecycle.rs — Implement daemon process management:

`pub async fn start_daemon()`:
- Check if daemon already running (read PID file, check if process alive via OpenProcess). If running, print "Daemon already running (pid: XXXX)" and exit 0.
- Create %LOCALAPPDATA%\wmux\ directory if not exists.
- Spawn a new detached process of the same binary with a hidden internal flag `--daemon-mode` (not exposed in clap help). Use `std::process::Command` with `.creation_flags(DETACHED_PROCESS | CREATE_NO_WINDOW)` on Windows to background it. The DETACHED_PROCESS flag (0x00000008) and CREATE_NO_WINDOW flag (0x08000000) ensure it survives terminal close.
- Print "Daemon started (pid: XXXX)" and exit.

`pub async fn run_daemon()`:
- This is the actual daemon main loop (called when --daemon-mode is active).
- Write PID file with current process ID.
- Initialize tracing to log file (append mode).
- Log "wmux daemon started" at info level.
- Enter a tokio::select! loop that listens for shutdown signal (will be expanded in Plan 02 to listen on Named Pipe). For now, just tokio::signal::ctrl_c() as placeholder.
- On shutdown: remove PID file, log "wmux daemon shutting down", exit cleanly.

`pub fn daemon_status()`:
- Read PID file. If not exists, print "Daemon is not running" and exit 0.
- Check if process with that PID is alive (OpenProcess + check). If alive, print "Daemon is running (pid: XXXX, uptime: unknown)". If dead (stale PID file), remove stale PID file and print "Daemon is not running (cleaned stale PID file)".

`pub fn kill_server()`:
- Read PID file. If not exists, print "Daemon is not running" and exit 0.
- Open process handle via OpenProcess, terminate via TerminateProcess.
- Remove PID file.
- Print "Daemon stopped".

Update src/main.rs to:
- Add a hidden `#[arg(long, hide = true)] daemon_mode: bool` flag to Cli struct (or use a hidden subcommand `_daemon-run`).
- Before matching commands, check if daemon_mode is set — if so, call `run_daemon()` directly.
- Wire `DaemonStart` -> `lifecycle::start_daemon()`, `Status` -> `lifecycle::daemon_status()`, `KillServer` -> `lifecycle::kill_server()`.

Use windows-rs for process management:
- `windows::Win32::System::Threading::{OpenProcess, TerminateProcess, GetCurrentProcessId, PROCESS_TERMINATE, PROCESS_QUERY_LIMITED_INFORMATION}`
- `windows::Win32::Foundation::{CloseHandle, HANDLE}`
